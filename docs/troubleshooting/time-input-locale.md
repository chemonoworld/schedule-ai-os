# 시간 입력 AM/PM 로케일

## 증상

Task 수정 모달의 시간 입력 필드에서 AM/PM 표시가 앱 언어 설정과 무관하게 표시됨.

## 원인

`<input type="time">`은 브라우저 네이티브 컨트롤로, 앱의 i18n 설정이 아닌 **시스템 로케일**을 따름.

- macOS 시스템 언어가 한국어 → "오전/오후"
- macOS 시스템 언어가 영어 → "AM/PM"

## 결론

**정상 동작**. 브라우저 네이티브 time picker는 OS 로케일을 따르므로 앱 레벨에서 제어 불가.

## 대안 (필요시)

커스텀 time picker 컴포넌트 구현:
- react-datepicker
- react-time-picker
- 직접 구현

현재는 네이티브 컨트롤 유지 (UX 일관성, 구현 복잡도 고려).
